# 创建管理员账号操作指南

## 概述

本文档详细说明了如何在家教接单平台中自定义创建管理员账号。系统支持通过云开发控制台直接操作数据库来创建管理员账号，也可以通过管理员后台界面添加新的管理员。

## 前置条件

- 已部署 `adminLogin` 和 `initAdminData` 云函数
- 云开发环境ID: `cloud1-3gqdqvkpbeab224c`
- 具有云开发控制台访问权限

## 方法一：通过云开发控制台创建

### 1. 访问云开发控制台

1. 打开微信开发者工具
2. 进入云开发控制台
3. 选择数据库模块
4. 找到 `admin_config` 集合

### 2. 手动添加管理员记录

在 `admin_config` 集合中添加新记录，包含以下字段：

```json
{
  "username": "自定义用户名",
  "password": "自定义密码",
  "nickName": "管理员显示名称",
  "avatarUrl": "头像URL（可选）",
  "email": "邮箱地址（可选）",
  "phone": "手机号码（可选）",
  "permissions": [
    "user_manage",
    "order_manage", 
    "forum_manage",
    "system_manage"
  ],
  "status": "active",
  "createTime": "2024-01-01T00:00:00.000Z",
  "lastLoginTime": null,
  "createdBy": "manual"
}
```

### 3. 字段说明

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `username` | String | 是 | 管理员登录用户名，必须唯一 |
| `password` | String | 是 | 登录密码（明文存储，生产环境建议加密） |
| `nickName` | String | 是 | 管理员显示名称 |
| `avatarUrl` | String | 否 | 头像URL地址 |
| `email` | String | 否 | 邮箱地址 |
| `phone` | String | 否 | 手机号码 |
| `permissions` | Array | 是 | 权限列表，详见权限说明 |
| `status` | String | 是 | 账号状态：`active`（激活）或 `inactive`（禁用） |
| `createTime` | Date | 是 | 创建时间 |
| `lastLoginTime` | Date | 否 | 最后登录时间 |
| `createdBy` | String | 是 | 创建者标识 |

### 4. 权限列表说明

可用的权限包括：

- `user_manage`: 用户管理权限
  - 查看用户列表
  - 编辑用户信息
  - 启用/禁用用户账号
  
- `order_manage`: 订单管理权限
  - 发布新订单
  - 编辑订单信息
  - 删除订单
  - 查看订单统计
  
- `forum_manage`: 论坛管理权限
  - 管理帖子内容
  - 屏蔽/恢复帖子
  - 删除不当内容
  
- `system_manage`: 系统管理权限
  - 修改系统设置
  - 数据备份与恢复
  - 添加新管理员
  - 修改管理员密码

## 方法二：通过管理员后台创建

### 1. 登录管理员后台

1. 使用现有管理员账号登录小程序
2. 进入管理员后台页面
3. 切换到"系统管理"标签页

### 2. 添加新管理员

1. 点击"添加管理员"按钮
2. 在弹出的对话框中输入用户ID
3. 点击"确认"按钮完成添加

**注意**: 此方法需要目标用户已在系统中注册，通过用户ID将普通用户提升为管理员。

## 方法三：通过云函数批量创建

### 1. 创建自定义云函数

创建一个新的云函数 `createAdmin`：

```javascript
// cloudfunctions/createAdmin/index.js
const cloud = require('wx-server-sdk')

cloud.init({
  env: 'cloud1-3gqdqvkpbeab224c'
})

exports.main = async (event, context) => {
  const db = cloud.database()
  const { adminData } = event
  
  try {
    // 检查用户名是否已存在
    const existingAdmin = await db.collection('admin_config')
      .where({ username: adminData.username })
      .get()
    
    if (existingAdmin.data.length > 0) {
      return {
        success: false,
        message: '用户名已存在'
      }
    }
    
    // 创建管理员记录
    const result = await db.collection('admin_config').add({
      data: {
        username: adminData.username,
        password: adminData.password,
        nickName: adminData.nickName || '管理员',
        avatarUrl: adminData.avatarUrl || '',
        email: adminData.email || '',
        phone: adminData.phone || '',
        permissions: adminData.permissions || [
          'user_manage',
          'order_manage',
          'forum_manage'
        ],
        status: 'active',
        createTime: new Date(),
        lastLoginTime: null,
        createdBy: 'api'
      }
    })
    
    return {
      success: true,
      message: '管理员创建成功',
      adminId: result._id
    }
  } catch (err) {
    return {
      success: false,
      message: '创建失败',
      error: err.message
    }
  }
}
```

## 管理员登录测试

创建管理员账号后，可以通过以下步骤测试登录：

1. 打开小程序登录页面
2. 点击"管理员入口"按钮
3. 输入创建的用户名和密码
4. 点击"登录"按钮
5. 验证是否成功跳转到管理员后台

## 安全建议

### 1. 密码安全

- 使用强密码（至少8位，包含字母、数字、特殊字符）
- 定期更换管理员密码
- 生产环境中应对密码进行加密存储

### 2. 权限控制

- 根据实际需要分配最小权限
- 定期审查管理员权限
- 及时禁用离职人员账号

### 3. 账号管理

- 为每个管理员设置唯一的用户名
- 记录管理员操作日志
- 定期检查管理员登录记录

## 常见问题

### 1. 用户名重复

**问题**: 创建管理员时提示用户名已存在
**解决**: 更换唯一的用户名，或检查现有管理员列表

### 2. 登录失败

**问题**: 管理员无法登录
**解决**: 
- 检查用户名和密码是否正确
- 确认账号状态为 `active`
- 查看云函数日志排查错误

### 3. 权限不足

**问题**: 管理员登录后无法访问某些功能
**解决**: 检查管理员的 `permissions` 字段，确保包含所需权限

### 4. 数据库连接失败

**问题**: 云函数执行失败
**解决**:
- 检查云开发环境ID是否正确
- 确认云函数已正确部署
- 查看云开发控制台的错误日志
