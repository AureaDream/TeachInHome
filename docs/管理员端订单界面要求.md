# 家教小程序管理员端开发要求

## 页面布局结构

### 顶部区域
- 发布订单按钮
  - 固定位置显示在页面顶部
  - 按钮样式突出，便于管理员快速操作

### 管理订单区域
- 筛选功能区
  - 状态筛选器
  - 科目筛选器
- 订单列表展示区
  - 以卡片形式展示订单信息
  - 支持分页加载，优化性能

## 功能实现逻辑

### 发布订单功能

#### 数据收集逻辑
```
点击发布按钮 → 显示订单发布模态框 → 填写表单数据 → 验证必填字段 → 组装订单数据 → 调用云函数 → 关闭模态框
```

#### 云数据库同步逻辑
- 将订单信息与云数据库中的 orders 集合同步
- 具体字段映射：
  - student_name → 学生姓名字段
  - subject → 科目分类字段
  - grade → 学生年级字段
  - hours_required → 课时要求字段
  - price_per_hour → 课时费用字段
  - contact_info → 联系方式字段
  - status → 初始状态设为 "pending"（待接单）
  - create_time → 订单创建时间戳
  - admin_id → 发布管理员ID

#### 数据提交逻辑
```
wx.cloud.database().collection('orders').add({
  data: {
    ...订单数据,
    create_time: new Date(),
    status: 'pending'
  }
}).then(() => {
  显示成功提示 → 刷新订单列表 → 重置表单
})
```

### 管理订单功能

#### 筛选查询逻辑
```
const queryCondition = {}

if (当前状态 !== '全部') {
  queryCondition.status = 当前状态值
}

if (当前科目 !== '全科') {
  queryCondition.subject = 当前科目值
}

wx.cloud.database().collection('orders')
  .where(queryCondition)
  .orderBy('create_time', 'desc')
  .get()
```

#### 订单列表渲染逻辑
```
云数据库查询结果 → 解析订单数据 → 格式化显示信息 → 渲染订单卡片

订单卡片显示字段：
- 订单编号：对应云数据库中的 _id 字段
- 学生信息：对应云数据库中的 student_info 字段
- 科目年级：组合显示 subject 和 grade 字段
- 状态标签：根据 status 字段显示不同颜色状态
- 发布时间：格式化 create_time 字段显示
```

#### 状态更新逻辑
```
点击状态操作按钮 → 确认操作 → 调用云数据库更新 → 刷新列表显示

云数据库更新操作：
wx.cloud.database().collection('orders')
  .doc(订单ID)
  .update({
    data: {
      status: 新状态值,
      update_time: new Date()
    }
  })
```

## 数据同步机制

### 实时数据同步
- 订单发布后立即同步到云数据库 orders 集合
- 状态变更实时更新云数据库中对应订单的 status 字段
- 筛选条件变化时重新从云数据库查询最新数据

### 性能优化逻辑
```
let page = 0
const pageSize = 10

onReachBottom → page++ → 查询下一批数据 → 拼接现有列表
```

### 错误处理逻辑
```
云数据库操作失败 → 捕获异常 → 显示错误提示 → 记录错误日志

数据一致性保证：
本地操作 → 云数据库确认 → 操作成功 → 更新本地状态
```

## 用户体验优化

### 加载状态管理
- 数据加载时显示骨架屏
- 操作过程中显示加载提示
- 空状态时显示友好提示

### 交互反馈
- 成功操作显示 toast 提示
- 重要操作需要二次确认
- 表单验证实时反馈

## 关键实现要点

1. 所有订单数据操作必须与云数据库中的 orders 集合保持同步
2. 状态筛选条件需要与云数据库中的 status 字段精确匹配
3. 科目筛选条件需要与云数据库中的 subject 字段精确匹配
4. 订单创建时间使用云数据库的服务器时间戳确保一致性
5. 分页查询需要合理设置 limit 和 skip 参数
6. 数据更新后需要及时刷新本地缓存和界面显示
