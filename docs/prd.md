# 项目名称：家教接单平台

## 1. 项目概述
### 1.1 项目背景
当前家教市场存在以下痛点：家长找老师困难；信息费用昂贵；新手教师入门门槛虚高；家教行业普及率低。项目核心解决新手教师的简历编辑问题，并建立教师交流平台。现有类似程序过于臃肿且缺乏在线交流功能，本项目将填补这一空白。

### 1.2 项目目标
主要业务目标：实现月交易量100-150单，聚焦福建省用户群体。  
独特价值点：为新手教师提供入职引导，打造专属家教教师的交流社区。  
短期目标（3个月）：实现小程序流畅运行的核心功能。  
长期目标（1年）：在家教圈内普及使用。

### 1.3 目标用户
主要用户：大学生兼职家教教师（18-26岁）。  
核心使用场景：长期学科辅导、特长教学场景。

## 2. 功能需求
### 2.1 核心功能
- 简历一键生成系统
- 智能匹配算法
- 身份认证功能
- 在线论坛功能
- 后台管理功能

### 2.2 功能详细描述
#### 2.2.1 简历一键生成系统
- 通过问卷形式收集教师信息（姓名、性别、学校、专业、户籍、证书、擅长科目、个人经验）
- 自动生成完整文字简历
- 简历长期存储于个人账户专属栏目
- 新用户引导流程优先引导完成简历创建

#### 2.2.2 智能匹配算法
- 接单列表支持多维度筛选（学科/距离/报酬）
- 集成腾讯地图API显示订单与教师的实时距离
- 支持教师主动搜索订单

#### 2.2.3 身份认证功能
- 管理员通过特殊密码注册
- 管理员可生成随机身份码分发给教师
- 认证教师获得论坛发帖权限
- 界面区分认证/未认证教师身份标识

#### 2.2.4 在线论坛功能
- 认证教师可发帖交流（支持匿名模式）
- 所有用户可浏览/评论内容
- 帖子按时间倒序排列

#### 2.2.5 后台管理功能
- 管理员查看/修改用户简历
- 论坛内容管理（删除/置顶帖子）
- 订单发布与管理界面

### 2.3 用户流程
1. 管理员发布家教需求订单
2. 教师查看订单列表（系统推荐+自主筛选）
3. 教师截图订单信息并通过微信联系管理员
4. 管理员审核接单请求
5. 成功接单教师获得身份码
6. 认证教师可在论坛发帖交流

## 3. 非功能需求
### 3.1 性能要求
- 日均论坛评论量支持：30+
- 高峰期并发用户支持：50+
- 最大响应时间：≤2秒

### 3.2 安全要求
- 用户简历信息仅管理员可见
- 论坛仅显示用户昵称
- 不涉及支付功能与资金流转
- 无需实名认证

### 3.3 可用性要求
- 仅需适配微信小程序平台
- 核心操作路径：教师查看订单≤3次点击
- 新用户60秒内完成简历创建

## 4. 技术规格
### 4.1 技术栈
- 前端：微信小程序框架
- 后端：Node.js


### 4.2 系统架构
- 纯小程序端架构
- 云端数据存储
- 模块化组件设计

### 4.3 第三方集成
- 腾讯地图API（距离计算）

## 5. 交付物

- 首版上线时间：1周内

### 5.2 时间线
